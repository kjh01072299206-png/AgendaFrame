# 실데이터 매체 표본과 수집 원칙

## 표본

AgendaFrame의 분석 대상은 **22개 주요 중앙언론의 온라인 뉴스 표본**이다.

| 매체 유형 | 수 | 매체 |
| --- | ---: | --- |
| 종합일간지 | 10 | 경향신문, 국민일보, 동아일보, 문화일보, 서울신문, 세계일보, 조선일보, 중앙일보, 한겨레, 한국일보 |
| 경제매체 | 9 | 매일경제, 한국경제, 머니투데이, 서울경제, 아시아경제, 이데일리, 파이낸셜뉴스, 헤럴드경제, 조선비즈 |
| 뉴스통신사 | 3 | 연합뉴스, 뉴시스, 뉴스1 |

이 표본은 한국 언론 전체나 모든 온라인 뉴스를 대표하는 확률표본이 아니다. 전국 단위의 공적 의제 생산에 지속적으로 참여하는 주요 중앙언론을 목적표집한 비교 범위이며, 서비스와 보고서에는 실제 수집 성공 매체 수와 본문 분석 가능 매체 수를 별도로 표시한다.

지상파·종합편성채널·보도전문채널은 제외한다. 방송사의 온라인 기사만으로 실제 TV 편성 순서, 앵커 멘트, 영상·자막, 리포트 길이를 관측할 수 없고 신문 홈페이지 배치와 같은 척도로 비교하면 제작 방식의 차이를 프레임 차이로 오인할 수 있기 때문이다. 향후 방송 대본과 편성 데이터를 정식으로 확보하기 전에는 방송 프레이밍 분석을 제공하지 않는다.

매체의 진보·중도·보수 성향을 사전 라벨로 부여하지 않는다. 결과는 매체별·이슈별 근거로 제시하며, 조선일보와 조선비즈처럼 동일 미디어그룹에 속한 매체는 그룹 정보를 함께 기록해 독립 출처 수를 해석할 때 중복 가능성을 밝힌다.

## 수집 경로

기사 목록의 기본 경로는 관리자가 이용 조건을 확인해 확보한 BigKinds Excel·CSV 또는 같은 형식의 메타데이터 파일이다. 홈페이지 배치는 별도 수집기가 같은 시각·뷰포트 정책으로 관측해 보호된 API로 전송할 수 있다. Sites Worker의 정기·전면 크롤링은 비활성화하며, 관리자 실행 시 이미 등록된 공식 기사 URL을 최대 20건씩 제한 요청해 본문을 메모리에서만 분석한다.

- 저장 대상: 기사 식별자, 제목, 원문 URL, 매체, 매체 유형·그룹, 섹션, 게시 시각, 수집 시각
- 분석 결과 저장: 프레임 유형·점수·기사 ID·임시 본문 기반 여부와 원문 링크
- 저장 제외: 자동 분석한 기사 전문·HTML·원문 문장, 기사 이미지, 회원 정보, 댓글
- 중복 방지: 공급자 기사 식별자와 정규화된 원문 URL을 각각 고유 키로 사용
- 활성화 조건: 관리자 인증과 CSV 형식·공식 도메인·시각 검증을 모두 통과한 경우에만 저장
- 직접 크롤링: 정기·전면 수집은 전 매체 비활성화. 관리자 실행의 공식 URL 제한 분석만 허용하며 로그인·유료벽·접근 차단은 우회하지 않음

가져오기 형식은 `source,title,url,published_at,collected_at,section,homepage_placement,homepage_rank`이다. 관리 화면은 최대 20,000행을 읽어 API에 500행씩 전송하며, 원문 URL은 등록된 22개 매체의 공식 HTTPS 도메인만 허용한다. 동일 URL은 다시 저장하지 않고 중복 건수로 기록한다.

## 홈페이지 배치의 한계

관리자 또는 승인된 수집기가 실제 홈페이지를 관측한 경우에만 TOP·MAIN·SECTION·LIST 배치를 저장한다. 각 관측은 시각, 뷰포트, 좌표, 카드 크기, 화면 상단 여부, 모듈, 순위를 별도 행으로 남긴다. 확인하지 않은 배치 값은 추정하거나 임의로 채우지 않고 `null`로 저장한다.

의제 분석은 실제 기사 메타데이터와 반복 배치 관측에 대해서만 실행한다. 홈페이지 배치가 없으면 고정 중립값을 넣지 않고 해당 성분을 제외한 뒤 관측된 성분의 가중치를 다시 계산한다. 프레임 단서 분석은 임시 본문을 읽을 수 있으면 본문을 우선하고, 없으면 제목으로 제한한다. 본문은 분석 직후 폐기하고 공급자·모델·클러스터링·점수·프레임 분류 버전과 `body_transient` 근거 범위만 결과에 저장한다.
