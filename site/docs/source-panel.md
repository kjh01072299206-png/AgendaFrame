# 실데이터 매체 표본과 수집 원칙

## 표본

AgendaFrame의 1차 실데이터 표본은 전국 종합일간지 5곳을 `2 : 1 : 2`로 구성한다.

| 분석상 표본 위치 | 매체 |
| --- | --- |
| 진보 표본 | 한겨레, 경향신문 |
| 중도 표본 | 한국일보 |
| 보수 표본 | 중앙일보, 조선일보 |

이 분류는 매체의 영구적이거나 객관적인 속성을 판정하는 라벨이 아니다. 기존 연구에서 반복적으로 사용된 비교 구도를 표본 추출에 적용한 것이며, 실제 결과는 매체별·이슈별로 제시한다. 집단 평균은 보조 지표로만 사용한다.

모두 같은 유형의 전국 종합일간지로 맞춰 방송사·통신사·경제지의 제작 방식 차이가 정치적 차이로 오인될 가능성을 줄인다. 각 매체는 같은 수집 시간대와 같은 최대 기사 수를 적용하고, 매체별 동일 가중치를 사용한다.

## 수집 경로

기사 목록의 기본 경로는 관리자가 BigKinds에서 확보한 Excel·CSV 파일이다. 홈페이지 배치는 별도 수집기가 같은 시각·뷰포트 정책으로 관측해 보호된 API로 전송할 수 있다. 배포된 Sites Worker 자체는 언론사 사이트를 직접 호출하지 않는다.

- 저장 대상: 기사 식별자, 제목, 원문 URL, 매체, 섹션, 게시 시각, 수집 시각
- 조건부 저장: 이용 권한·분석 허용·공개 근거 허용 여부가 기록된 기사 본문
- 저장 제외: 권한이 확인되지 않은 기사 본문, 기사 이미지, 회원 정보, 댓글
- 중복 방지: 공급자 기사 식별자와 정규화된 원문 URL을 각각 고유 키로 사용
- 활성화 조건: 관리자 인증과 CSV 형식·URL·시각 검증을 모두 통과한 경우에만 저장
- 직접 크롤링: 전 매체 비활성화

가져오기 형식은 `source,title,url,published_at,collected_at,section,homepage_placement,homepage_rank`이다. 관리 화면은 최대 20,000행을 읽어 API에 500행씩 전송하며, 원문 URL은 해당 언론사의 HTTPS 도메인만 허용한다. 동일 URL은 다시 저장하지 않고 중복 건수로 기록한다.

## 홈페이지 배치의 한계

관리자 또는 승인된 수집기가 실제 홈페이지를 관측한 경우에만 TOP·MAIN·SECTION·LIST 배치를 저장한다. 각 관측은 시각, 뷰포트, 좌표, 카드 크기, 화면 상단 여부, 모듈, 순위를 별도 행으로 남긴다. 확인하지 않은 배치 값은 추정하거나 임의로 채우지 않고 `null`로 저장한다.

의제 분석은 실제 기사 메타데이터와 반복 배치 관측에 대해서만 실행한다. 홈페이지 배치가 없으면 고정 중립값을 넣지 않고 해당 성분을 제외한 뒤 관측된 성분의 가중치를 다시 계산한다. 프레임 단서 분석은 승인 본문이 있으면 본문을 우선하고, 없으면 제목으로 제한한다. 공급자·모델·클러스터링·점수·프레임 분류 버전과 근거 범위를 결과에 함께 저장한다.
